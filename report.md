# Gold Price Forecaster: Machine Learning App for Commodity Price Prediction

## 1. Name and Purpose of the App

**Name:** Gold Price Forecaster
**Purpose:** This application predicts the next day's closing price of Gold. By leveraging historical price data and engineered financial indicators, the app aims to assist traders and analysts in identifying short-term price movements and trends in the gold commodities market.

## 2. Algorithms Used

The primary algorithm applied is the **Random Forest Regressor**. 
This ensemble machine learning algorithm was chosen because it can effectively capture complex, non-linear relationships in financial time-series data without requiring extensive hyperparameter tuning. It is also highly robust against overfitting compared to individual decision trees.

## 3. Dataset Information

*   **3.1 Dataset source/link:** Historical data is retrieved programmatically via the Yahoo Finance API (`yfinance` library), specifically targeting the Gold Futures ticker (`GC=F`).
*   **3.2 Number of records:** Approximately 1,250 rows (representing 5 years of daily trading data).
*   **3.3 Number of features:** 9 engineered features used for training.
*   **3.4 Description of features:**
    *   **Open:** (Float) The opening price of the trading day.
    *   **High:** (Float) The highest price during the trading day.
    *   **Low:** (Float) The lowest price during the trading day.
    *   **Close:** (Float) The closing price of the trading day.
    *   **Volume:** (Integer) Total number of contracts traded.
    *   **SMA_5:** (Float) 5-day Simple Moving Average.
    *   **SMA_20:** (Float) 20-day Simple Moving Average.
    *   **Daily_Return:** (Float) Percentage change in the closing price from the previous day.
    *   **Volatility_5:** (Float) Rolling 5-day standard deviation of the daily returns.
*   **3.5 Preprocessing steps:** 
    *   Downloaded continuous data while dropping days with missing (NaN) values due to market closures.
    *   Engineered financial indicators corresponding to both short-term trends (SMA_5) and momentum (Volatility).
    *   Shifted the `Close` price backwards by one step (`shift(-1)`) to create the predictive target label (`Target_Next_Close`).
    *   Performed a final row-drop strategy to omit NaN values generated by the rolling windows and shifting logic.

## 4. Libraries, Toolkits, and Frameworks

*   **Pandas:** Handling the data structures, indexing time-series data, and feature engineering (rolling windows, shifts).
*   **NumPy:** Providing underlying array operations and support for complex calculations.
*   **Scikit-Learn (sklearn):** Providing the regression models (`RandomForestRegressor`), data splitting logic (`train_test_split`), and evaluation metrics.
*   **Matplotlib & Seaborn:** Generating the visualization to compare the actual gold prices against the model's predictions.
*   **yfinance:** Downloading reliable and up-to-date historical open dataset values continuously directly from Yahoo Finance without maintaining local CSV files.

## 5. Application Design and Implementation

The application operates as a linear data pipeline:
1.  **Data Flow:** The application first calls `yfinance` to request the last 5 years of daily gold metrics. The response is flattened into a one-dimensional Pandas DataFrame.
2.  **Feature Application:** Moving averages and daily yield volatility constraints are calculated row-by-row. The target column is formed using a negative shift algorithm.
3.  **Model Training:** The time-series data is split 80/20 chronologically (ensuring that future data is not leaked into the training set). The Random Forest model is trained on the first 80%.
4.  **Prediction:** The trained algorithm runs against the unseen 20% test partition to measure validity, and then finally ingests the most recent single row to predict tomorrow's closing limit price.

## 8. Discussion and Insights

**Performance:** The Random Forest algorithm performed well at tracking mid-to-long term trends in the market. Since financial data contains elements of randomness, the model appropriately leverages smoothed out features (like SMA) to avoid extreme predictive shifts.
**Limitations:** The model exhibits slight latency/lag during extremely sharp, unprecedented volatility spikes (e.g., sudden global economic news). Random Forest is inherently a bounding algorithm; it cannot predict a price higher than the highest observed training price. 
**Potential Improvements:** 
*   Transitioning from a tree-based ensemble method to an LSTM (Long Short-Term Memory) Recurrent Neural Network would allow the context of sequential time jumps to be respected mathematically.
*   Incorporating macroeconomic external datasets (e.g., interest rates, inflation markers, USD index) would significantly expand the predictive horizon.
